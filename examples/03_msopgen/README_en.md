# msopgen Custom Operator Project Generation Example

## Overview

This example demonstrates how to use the `msopgen` tool to generate a custom operator project template based on the AddCustom operator, and verifies the generated custom operator through an ACLNN invocation example.

`msopgen` is an Ascend AI custom operator development tool that generates a custom operator project framework from an operator prototype definition file, including the CMake build system, Host-side code, Kernel-side code, and framework adaptation layer.

## Supported Products and CANN Software Versions

| Product | CANN Software Version |
|------|-------------|
| Ascend 950PR/Ascend 950DT | >= CANN 9.1.0 |
| Atlas A3 Training Series Products/Atlas A3 Inference Series Products | >= CANN 9.0.0 |
| Atlas A2 Training Series Products/Atlas A2 Inference Series Products | >= CANN 9.0.0 |

## Directory Structure

### Directory Structure Before msopgen Generation

```shell
├── aclnn_invocation                // ACL NN API invocation example for verifying the custom operator
└── op_dev                          // Operator development directory, containing operator source code
    ├── add_custom.json             // Operator prototype definition file (input/output description)
    ├── op_host                     // Host-side code directory
    │   └── add_custom.cpp          // Tiling implementation and operator definition registration
    └── op_kernel                   // Kernel-side code directory
        ├── add_custom.cpp          // Kernel function implementation
        └── add_custom_tiling.h     // Tiling data structure definition
```

### Directory Structure After msopgen Generation

```shell
├── aclnn_invocation                // ACL NN API invocation example (unchanged)
├── op_dev                          // Operator development directory, containing operator source code
└── custom_op                       // Custom operator project directory generated by msopgen
    ├── build.sh                    // Build script
    ├── CMakeLists.txt              // CMake configuration
    ├── CMakePresets.json           // CMake preset configuration
    ├── framework                   // Framework adaptation layer
    │   ├── CMakeLists.txt
    │   └── tf_plugin               // TensorFlow plugin
    │       ├── CMakeLists.txt
    │       └── tensorflow_add_custom_plugin.cc
    ├── op_host                     // Host-side code
    │   ├── add_custom.cpp          // Copied host-side operator implementation file, users can modify and replace
    │   └── CMakeLists.txt
    └── op_kernel                   // Kernel-side code
        ├── add_custom.cpp          // Copied kernel-side operator implementation file, users can modify and replace
        ├── add_custom_tiling.h     // Copied Tiling data structure definition, users can modify and replace
        └── CMakeLists.txt
```

## aclnn_invocation Description

The `aclnn_invocation` directory in this example provides a complete example of invoking a custom operator using the ACLNN API. This example is identical to the one in the [asc-devkit repository](https://gitcode.com/cann/asc-devkit/tree/master/examples/01_simd_cpp_api/02_features/01_invocation/aclnn_invocation), demonstrating how to load and execute a custom operator via the ACLNN API.

For more ACLNN API invocation examples and detailed instructions, please refer to: [ACLNN API Invocation Examples](https://gitcode.com/cann/asc-devkit/tree/master/examples/01_simd_cpp_api/02_features/99_acl_based/01_acl_invocation/aclnn_invocation)

## Build and Run

### Environment Preparation

Execute the following steps in the root directory of this example to build and run it.

- Configure environment variables

  Please configure environment variables according to the [installation method](../../docs/00_quick_start.md#prepare&install) of the CANN development toolkit on your current environment.

  ```bash
  source ${install_path}/cann/set_env.sh
  ```

  > **Note:** `${install_path}` is the CANN package installation directory. When no installation directory is specified, the default installation path is `/usr/local/Ascend`.

### Build and Execution Steps

Execute the following steps in the root directory of this example to run it.

1. **Generate the custom operator project**

    ```bash
    msopgen gen -i ./op_dev/add_custom.json -f <framework> -c ai_core-<soc_version> -lan cpp -out ./custom_op
    ```

    - soc_version: Ascend AI processor model. If the specific soc_version cannot be determined, run the `npu-smi info` command on the server with the Ascend AI processor installed to query it. Prepend "Ascend" to the queried "Name" value. For example, if the "Name" value is xxxyy, the actual soc_version to configure is Ascendxxxyy.
    - framework: Framework type. The default is TensorFlow, with default values: tf or tensorflow. For other available options, refer to the [msopgen User Guide](https://gitcode.com/Ascend/msopgen/blob/master/docs/zh/user_guide/msopgen_user_guide.md)
    - If the msopgen command is run multiple times to generate the operator project, please delete the existing custom_op directory first.

2. **Copy source code to the generated directory**

    ```bash
    cp -rf ./op_dev/op_kernel custom_op
    cp -rf ./op_dev/op_host custom_op
    ```

3. **Build the custom operator package**

    ```bash
    cd custom_op
    bash build.sh
    ```

4. **Install the custom operator package**

    ```bash
    cd build_out
    ./custom_opp_<target_os>_<target_architecture>.run
    ```

5. **Build and run the verification program**

    ```bash
    cd ../../aclnn_invocation
    mkdir -p build
    cd build
    cmake .. && make -j
    ./execute_add_op
    ```

6. **The execution result is as follows, indicating successful execution**

    ```log
    test pass
    ```
